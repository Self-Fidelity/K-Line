# 下一步开发计划

本文档记录项目的下一步开发方向和计划。

## 已确定的功能方向

### 1. 策略比较功能（待实现）
- **功能描述**：同时对比多个策略的表现
- **主要功能**：
  - 在同一图表中显示不同策略的信号
  - 对比收益率、胜率、最大回撤等指标
  - 生成策略对比报告
- **优先级**：高
- **状态**：待实现

### 2. 批量股票分析（待实现）
- **功能描述**：批量分析多只股票
- **主要功能**：
  - 支持批量输入股票代码列表
  - 扫描并筛选表现较好的股票
  - 生成批量分析报告
- **优先级**：中
- **状态**：待实现

### 3. 参数优化功能（待实现）
- **功能描述**：自动寻找最优策略参数
- **主要功能**：
  - 网格搜索或优化算法
  - 输出最优参数组合及回测结果
- **优先级**：中
- **状态**：待实现

### 4. 策略回测可视化增强（待实现）
- **功能描述**：增强策略回测的可视化效果
- **主要功能**：
  - 绘制资金曲线图
  - 显示持仓变化
  - 展示收益率曲线
- **优先级**：低
- **状态**：待实现

### 5. 策略组合功能（待实现）
- **功能描述**：组合多个策略的信号
- **主要功能**：
  - 加权投票机制
  - 提高信号准确性
- **优先级**：中
- **状态**：待实现

## WebUI 开发计划（新）

### 需求概述
开发一个 Web 用户界面，方便用户使用 K-Line 工具的各项功能。

### 技术选型（待讨论）
- **后端框架**：Flask / FastAPI / Django
- **前端框架**：原生HTML+JavaScript / React / Vue
- **图表库**：Bokeh Server / Plotly / ECharts

### 主要功能模块（待细化）
1. **数据管理**
   - 股票数据获取和更新
   - 数据查询和浏览
   
2. **策略分析**
   - 策略选择和配置
   - 策略执行和结果展示
   - 策略比较
   
3. **可视化**
   - K线图展示
   - 策略信号标记
   - 统计指标展示
   
4. **报告导出**
   - 数据导出
   - 图表导出
   - 策略报告生成

### 实现步骤（待讨论）
1. 确定技术栈
2. 设计UI/UX
3. 后端API开发
4. 前端界面开发
5. 功能集成和测试
